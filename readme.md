# Project Setup

This project uses Docker Compose to start a MySQL container and a Jupyter container.

## Requirements

- Docker Desktop installed and running
- `docker compose` available in your terminal
- The `.env` file configured with your database credentials

## Start the containers

From the project root, run:

```bash
docker compose up -d
```

If you need to reset everything and start fresh, run:

```bash
docker compose down -v
docker compose up -d
```

## Find the Jupyter link

After the containers start, check the Jupyter container logs:

```bash
docker logs mysql-group10-jupyter
```

Look for the local URL that starts with something like:

```text
http://127.0.0.1:8888/?token=...
```

Copy that link into your browser to open Jupyter.

## Open the notebook

Inside Jupyter, open the notebook named:

```text
usage_example.ipynb
```

This notebook contains the example workflow for connecting to the database, and seeding it using `proj2-fillSchema.py`.

## Notes

- MySQL initialization scripts in `./mysql/init` run only when the database volume is created for the first time.
- If changes to the schema do not appear, remove the volume with `docker compose down -v` and start again.
- Inside the Jupyter container, the MySQL host should be `mysql`.
- There is a schema svg located at `./work/schema.svg` generated by the schema notebook.
